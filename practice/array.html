配列

前回は、「選択構造」について勉強したね。
今回、「反復構造」をやるよ…って言いたいところだけど、反復構造の前に、データを格納する別のものを紹介するよ。

今まで学習した「変数」は、変数1つにつき、データも1つしか保存しておくことはできなかったね。
もし、もっとたくさんのデータを扱わなければならなくなったら、変数をたくさん用意しなきゃいけないね。
でも、それじゃ複雑になってしまうし、全部のデータに対して同じ処理をしようってなった時にとても不便に感じると思う。
そこで登場するのが、「配列」だよ。
「配列」は、たくさんのデータを効率よく扱うことに長けているんだ。
下の図を見てみて。


配列は、1つの変数をたくさんの部屋に区切ったものなんだ。
部屋1つ1つに、変数を代入することができるよ。
早速、使い方をやっていくよ。 

#include <stdio.h>
int main(){
  // 配列の宣言
  int a[10];

  return(0);
}

普通の変数の宣言と似ているね。違いは、変数名の後ろに「[10]」がついていることかな。
配列は、「変数をいくつかの部屋に区切るもの」ということから、変数aを10個の部屋に区切ってそうだね。
これだけだと、よく分からないから、配列の中に代入してみよう。 

#include <stdio.h>
int main(){
  // 配列の宣言
  int a[10];

  // 値の代入
  a[1] = 1;
  a[2] = 2;

  return(0);
}

ソースコードの7行目・8行目をみて分かる通り、「変数名[番号]」で、配列の中身を参照できるみたい。
今まで通り、出力も試してみよう。 

#include <stdio.h>
int main(){
  // 配列の宣言
  int a[10];

  // 値の代入
  a[1] = 1;
  a[2] = 2;

  // 変数の中身を出力
  printf("a[1] = %d\n", a[1]);
  printf("a[2] = %d\n", a[2]);

  return(0);
}

うまくいったかな？
変数の宣言の仕方、変数の中の部屋の参照ができるようになったね!!

ところで、今回は10個の部屋の配列を作ったよね。じゃあ、配列の10番目がある、ということかな。
配列の宣言・参照のやり方は学習したから…下のようなソースができるね。
でも、実行はしないでね。 

#include <stdio.h>
int main(){
  // 配列の宣言
  int a[10];

  // 値の代入
  a[10] = 8;

  // 変数の中身を出力
  printf("a[10] = %d\n", a[10]);

  return(0);
}

実は、変数を宣言すると、例えば10個の部屋の変数を宣言した場合、「0〜9の10部屋」が作られるんだ。
つまり、「10番目の部屋」は存在しないんだ。
このように「配列の外を参照してしまう」ことを「配列外参照」というよ。(そのままじゃん)
配列外参照を起こすと、何が起こるかわからないんだ。
一見、正常に動作しているように見えることさえあるんだ。
プログラミングに慣れていてもやりがちなミスだから、配列を扱うときは気をつけよう。

多次元配列

配列の使い方が割とわかってきたかな？
ここからは、競技プログラミングでよく使うけれど、少しだけ難易度が高いことをするから、配列が慣れてからチャレンジしてみてもいいよ。

配列は、用途に応じて何次元でも宣言できるんだ。
「多次元配列」っていうよ。

#include <stdio.h>
int main(){
  // 配列
  int a[10];

  // 2次元配列
  int b[10][10];

  // 5次元配列
  int c[10][10][10][10][10];

  return(0);
}

こんな感じで、何次元に増えても同じように宣言できるよ。
でも、現実的に人間が頭の中で想像できるものは「３次元」くらいまでなので、四次元以上になってしまう場合は「構造体」というものを使うといいよ。(時間があったら説明書きます。)

一応参照も。
今までと同じような形で参照できるよ。

#include <stdio.h>
int main(){
  // 配列の宣言
  int a[10][10];
  
  // 値の代入
  a[0][0] = 100;
  a[5][3] = 530;
  
  // 出力
  printf("a[0][0] = %d\n", a[0][0]);
  printf("a[5][3] = %d\n", a[5][3]);
   
  return(0); 
}

わかってきたかな？
配列も、条件式のように「使って慣れる」しかないから、ゆっくりでも慣れていこうね。
